This guide will get you up and running with gurl in just a few minutes. We'll cover the most common use cases to help you start making HTTP requests right away.

## Your First Request

<Steps>
  <Step title="Make a basic GET request">
    The simplest way to use gurl is to make a GET request to a URL:

    ```bash
    gurl request https://dummyjson.com/products
    ```

    **What to expect:** You'll see a JSON response containing a list of products. gurl automatically detects JSON responses and displays them in a readable format.
  </Step>

  <Step title="Format JSON output">
    For even prettier JSON formatting, use the `-j` flag:

    ```bash
    gurl request -j https://dummyjson.com/products/1
    ```

    **What to expect:** The JSON response will be formatted with proper indentation and syntax highlighting, making it easier to read complex responses.
  </Step>
</Steps>

## Send Data with POST Requests

<Steps>
  <Step title="POST JSON data">
    Send JSON data to an API endpoint using the `-X POST` method and `-d` flag:

    ```bash
    gurl request https://dummyjson.com/auth/login \
      -X POST \
      -H "Content-Type: application/json" \
      -d '{"username":"emilys","password":"emilyspass","expiresInMins":30}'
    ```

    **What to expect:** You'll receive an authentication response with a token. The `-H` flag sets custom headers, and `-d` provides the request body.

    <Tip>
      Use the `--json-request` flag as a shortcut to automatically set `Content-Type: application/json`
    </Tip>
  </Step>

  <Step title="Use authentication tokens">
    Include authentication headers in your requests:

    ```bash
    gurl request https://dummyjson.com/auth/me \
      -H "Authorization: Bearer YOUR_ACCESS_TOKEN_HERE"
    ```

    **What to expect:** The API will return user information based on the provided token.
  </Step>
</Steps>

## Download Files

<Steps>
  <Step title="Save response to a file">
    Save API responses or download files using the `-o` flag:

    ```bash
    gurl request -o response.json https://dummyjson.com/users
    ```

    **What to expect:** The response will be saved to `response.json` in your current directory instead of being displayed in the terminal.
  </Step>

  <Step title="Use the download command">
    For large files, use the dedicated download command with progress tracking:

    ```bash
    gurl download -o myfile.zip https://example.com/file.zip
    ```

    **What to expect:** You'll see a progress bar showing the download status, speed, and estimated time remaining.

    <Note>
      The download command supports resume capability with the `-c` flag and automatic retries with `--tries`
    </Note>
  </Step>
</Steps>

## Common Patterns

### Follow Redirects

Many APIs use redirects. Follow them automatically with the `-L` flag:

```bash
gurl request -L https://dummyjson.com/http/200
```

### Verbose Output

See detailed request/response information including headers and timing:

```bash
gurl request -v https://dummyjson.com/test
```

### Different HTTP Methods

Use any HTTP method with the `-X` flag:

<CodeGroup>
```bash POST
gurl request -X POST https://dummyjson.com/products
```

```bash PUT
gurl request -X PUT https://dummyjson.com/products/1
```

```bash DELETE
gurl request -X DELETE https://dummyjson.com/products/1
```
</CodeGroup>

## Advanced Download Features

### Resume a partial download

```bash
gurl download -c https://example.com/large-file.iso
```

### Download with retries

```bash
gurl download --tries 5 --retry-delay 5s https://example.com/file.zip
```

### Quiet mode (no progress bar)

```bash
gurl download -q https://example.com/file.zip
```

## Pro Tips

<CardGroup cols={2}>
  <Card title="Set Timeouts" icon="clock">
    Use `--timeout` to set a custom request timeout:
    ```bash
    gurl request --timeout 60s https://api.example.com
    ```
  </Card>

  <Card title="Ignore SSL Errors" icon="shield">
    Use `-k` for testing with self-signed certificates:
    ```bash
    gurl request -k https://localhost:8443
    ```
  </Card>

  <Card title="Wait Before Request" icon="hourglass">
    Add a delay before making the request:
    ```bash
    gurl request --wait-time 5s https://api.example.com
    ```
  </Card>

  <Card title="Control Redirects" icon="turn-down">
    Set maximum redirect limit:
    ```bash
    gurl request --max-redirects 5 https://api.example.com
    ```
  </Card>
</CardGroup>
